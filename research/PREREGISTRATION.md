# ПРЕ-РЕГИСТРАЦИЯ ГИПОТЕЗЫ — InsiderScanner v2
**Дата фиксации: 2026-06-10. После начала сбора этот файл НЕ редактируется.**

## Контекст
Бэктест данных марта-апреля 2026 (см. HLCarryBot/brain/insider_scanner_verdict.md):
broad-сигналы edge не имеют; единственный недоказанный срез — исполнимые лонгшоты ≥$700
(win 11.6% vs implied 5.6%, 1.7σ на 43 юнитах). Этот сбор — проверка ровно этого среза.

## H1 (основная гипотеза)
Сигналы, удовлетворяющие ВСЕМ условиям на момент детекта (поля из signals_enrich):
- `is_sports = 0`
- `exec_px_300 IS NOT NULL AND exec_px_300 < 0.10` (исполнимо $300 по ask дешевле 10¢)
- `ask_depth_12x_usd >= 200` (глубина в пределах 1.2× цены сделки)
- `usd_size >= 700`
- `enrich_status = 'ok'` и `detect_lag_s <= 300`
имеют win-rate, превышающий exec_px_300 (implied) с преимуществом ≥ 3σ (binomial)
И средний ROI > 0 при входе по exec_px_300, flat stake.

## Правила анализа
- Юнит = (gamma_slug, token side). Первый подходящий сигнал на юнит.
- Резолюция: Gamma outcomePrices (decisive: max > 0.99).
- Требуемый объём: **n ≥ 120 юнитов** (~3 мес. по темпу марта-апреля).
- Один промежуточный взгляд на n=60, порог 3.5σ. Других подглядываний нет.
- Окна с дырами heartbeat > 5 мин подряд исключаются целиком (день целиком при дыре > 1ч).

## Негативный контроль (обязателен)
Тот же критерий на срезе `usd_size < 700` (в бэктесте win 2.0% — заведомый шум).
Если «edge» ≥ 3σ появится и там → системная ошибка пайплайна, H1 не засчитывается.

## Критерий решения
- H1 подтверждена (≥3σ И ROI>0 И негативный контроль чист) → строить исполнитель, размер $20-30/юнит.
- Иначе → инсайдер-направление закрывается НАВСЕГДА, без переформулировок гипотезы задним числом.
